How a project thinks, remembers,
and stays auditable.

SOS gives a human and an AI agent a shared shape for project memory: a routed context surface, a vault for capture → triage → durable knowledge, and a gated command vocabulary that never overwrites without consent.

06

Operating principles

non-negotiable
P.01
File-first, template-first
Source of truth lives on disk as Markdown + manifests. The visual map and assistants summarize — they never override.
P.02
Install never overwrites
/sos:init proposes missing files first. Replacement, merge, or repair takes a separate explicit approval path.
P.03
One actor registry
.claude/ACTORS.md is canonical. Vault content may reference actors, but parallel rosters are forbidden.
P.04
Archive preserves bytes
Binary, encrypted, compressed, generated, and vendor files are copied byte-for-byte. No re-saving, no pretty-printing.
P.05
Routed surface, not piles
Required files are not enough — a fresh agent must find them. PM.md is the canonical map; adapters route into it.
P.06
Stronger tool can't mutate older node
If the project SOS version is newer than the running tool, writes stop. Ask the user to update first.
